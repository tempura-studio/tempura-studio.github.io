# mymap プライバシーポリシー

**最終更新日**: 2026-06-02 (完全ローカルモードを既定化、クラウドバックアップを opt-in 方式に変更)

mymap (以下「本アプリ」) は、ユーザーのプライバシーを尊重し、本アプリの利用にあたって取得する情報の取り扱いについて、本ポリシーで明らかにします。

本アプリは初期状態では取得した情報をすべて**お使いの端末内のみに保存**します。クラウドへの保存は、ユーザーが明示的に「クラウドバックアップ」を有効化したときに限り発生します。

## 1. 取得する情報

本アプリは、以下の情報を取得します。

| 情報の種別 | 取得タイミング | 利用目的 |
|---|---|---|
| 位置情報 (現在地・訪問座標) | アプリ起動中およびバックグラウンド | 地図表示・訪問エリアの記録機能の提供 |
| Apple ID / Google アカウントに紐づくメールアドレス | ユーザーがクラウドバックアップを有効化し、Apple / Google でサインインしたときのみ | アカウント識別・バックアップデータの引き継ぎ |
| ユーザー識別子 (Firebase UID) | ユーザーがクラウドバックアップを有効化したときのみ生成 | バックアップデータの保存先の分離 |
| 購入履歴 (App Store のレシート情報) | 有料機能を購入したとき | プレミアム機能の解放状態の管理 |

クレジットカード番号などの決済情報は、Apple および RevenueCat が処理し、本アプリには一切渡されません。

クラウドバックアップを有効化していない状態では、Apple ID / Google アカウント情報や Firebase UID は一切取得しません。端末内で動作する位置情報の記録と、購入履歴のみが扱われます。

**位置情報に関する重要事項**:
- 本アプリは訪問したエリアを **約 130m 直径の六角形タイル (H3 解像度 10)** の集合として記録します。元の GPS 座標そのものではなく、「どのタイルを訪れたか」という ID のみを保存します。
- 電車・自動車・新幹線・トンネル等で GPS の取得タイミングが間に合わなかった区間については、両端のタイルから端末側で推定した経路上のタイル ID を追加で保存します。
- 「**いつ訪れたか**」の時刻情報・順序情報は保存しません。記録の時系列や日時は端末にもクラウドにも残らないため、過去の行動を時系列で再構成することはできません。
- 端末内の保存データは「訪問したことのあるタイル、および移動経路上にあると推定されたタイルの ID 集合」であり、同じタイルに何度入っても 1 件としてのみ扱われます (同じ場所を周回しても記録は増えません)。

## 2. 情報の保存先

本アプリのデータ保存先は、ユーザーが「クラウドバックアップ」を有効化しているかどうかで変わります。

### 既定の状態 (完全ローカルモード)

クラウドバックアップを有効化していない状態 (アプリのインストール直後はこの状態です) では、データは以下の場所にのみ保存されます。

- **端末内 (IndexedDB / UserDefaults)**: 訪問タイル ID
- **決済プラットフォーム**: 購入履歴・プレミアム状態 (Apple App Store / RevenueCat)

この状態では、本アプリはクラウド認証サービス・クラウドストレージ・クラウドデータベースのいずれにも接続しません。位置情報は端末から外部に送信されません。

**注意**: 完全ローカルモードではアプリの削除や端末の故障でデータが失われます。引き継ぎが必要な場合はクラウドバックアップを有効化してください。

### クラウドバックアップを有効化した状態

ユーザーがメニュー > アカウント から「クラウドバックアップを有効化」を選択し、Apple / Google サインインを完了すると、以下の場所にもデータが保存されるようになります。

- **クラウド認証サービス**: Firebase UID、Apple / Google アカウントに紐づくメールアドレス
- **クラウドストレージ**: 訪問タイル ID の集合。タイル ID の先頭文字でグループ化した NDJSON ファイル (1 行 1 タイル ID) としてユーザー毎に分離して保存
- **クラウドデータベース**: プレミアム状態フラグ・最終アクセス日時 (アカウント自動削除判定用)

サーバーは Google Cloud を利用しています。具体的な事業者は Section 3 をご確認ください。

クラウドバックアップは「クラウドバックアップを停止」操作でいつでも停止できます。停止すると新規のクラウド送信は行われなくなりますが、それまでにクラウドに保存されたデータは保持されます (同じアカウントで再度有効化すれば復元されます)。完全に削除したい場合は Section 4 の「アカウントを削除」を行ってください。

**ユーザー操作によって記録に「穴」を作る方法**:
本アプリでは、以下の操作により特定区間の自動補間を停止できます。意図的に記録を残したくない区間がある場合にご利用ください。

- App Switcher (アプリスイッチャー) からアプリをスワイプで終了する → 次回起動時の最初の位置情報からは新規記録扱いとなり、その間の経路は補間されません。
- iOS の設定で本アプリの位置情報利用を OFF にする → 再 ON 後の最初の位置情報からも同様に新規記録扱いになります。

(なお、同じ場所に長時間滞在した場合に電池節約のために自動で発動する省電力モードは、上記の「穴を作る操作」とは異なり、移動再開後も補間を継続します。)

## 2.1 運営者によるアクセス

本アプリの運営者 (個人開発者) は、上記サービスの管理ダッシュボードから、技術的には全ユーザーのデータを閲覧することが可能です。ただし、以下の目的以外で閲覧することはありません。

- 障害調査・技術サポート (ユーザーからの問い合わせ対応時など)
- 統計集計 (個人を特定しない集計値の利用)

第三者への提供、広告利用、営利目的での個人データ解析は一切行いません。一般のユーザー同士のデータは、サーバー側のアクセス制御によってユーザー毎に分離されており、他のユーザーが閲覧することはできません。

## 3. 第三者提供

法令に基づく場合を除き、ユーザーの同意なく第三者に個人情報を提供することはありません。

本アプリは以下の第三者サービスを利用しています。各サービスのプライバシーポリシーは下記をご確認ください。

- [Apple - Sign in with Apple](https://www.apple.com/legal/privacy/)
- [Google - Firebase](https://firebase.google.com/support/privacy)
- [Google - Google アカウント](https://policies.google.com/privacy)
- [RevenueCat Privacy Policy](https://www.revenuecat.com/privacy/)

## 4. データの削除

### クラウドバックアップを有効化している場合

ユーザーは、アプリ内のメニュー > アカウント > 「アカウントを削除」から、ご自身のアカウントとすべての訪問データをいつでも完全に削除できます。

削除すると、以下がすべて完全に消去されます。

- クラウド認証サービス上のアカウント (Firebase UID とメールアドレス)
- クラウドストレージに保存された訪問タイル ID
- クラウドデータベース上のプレミアム状態
- 端末内のキャッシュデータ
- Apple ID 連携 (Apple サインインを利用している場合は revokeAccessToken により Apple 側の連携も解除されます)

削除後に同じアカウントで再度サインインしても、削除されたデータは復元されません (新しい UID が発行されます)。

### 完全ローカルモードの場合

完全ローカルモードでは、データは端末内のみに保存されています。iOS の「設定 > 一般 > iPhone ストレージ > mymap > App を削除」または App Store からのアプリ削除を行うと、本アプリが端末内に保存しているデータは完全に消去されます。

### その他

削除ボタンが操作できない、トラブルでアプリにアクセスできない等の場合は、下記の問い合わせ先までご連絡ください。受領後、合理的な期間内に対応します。

## 5. クッキーおよびトラッキング

本アプリは広告トラッキングを行いません。ATT (App Tracking Transparency) の許可ダイアログは表示されません。

## 6. お子様のプライバシー

本アプリは、13 歳未満のお子様による意図的な情報収集を行いません。

## 7. ポリシーの変更

本ポリシーは予告なく変更される場合があります。重要な変更がある場合は、アプリ内またはこのページで通知します。
